## Подготовка к тестированию

### 1. Создать скриншоты для ручных шагов

Создай папку `assets\` (если её нет) и добавь два скриншота:

```bash
mkdir assets
```

**Необходимые скриншоты:**

- `assets\step05_kiro_connect.png`
  - Страница: http://localhost:20128/providers
  - Что показать: Провайдер Kiro с кнопкой "Connect"
  - Желательно обвести кнопку красной рамкой или стрелкой

- `assets\step06_create_api_key.png`
  - Страница: http://localhost:20128/api-keys
  - Что показать: Кнопку "Create API Key" и окно с созданным ключом
  - Желательно показать, где копировать ключ

**Как создать скриншоты:**
1. Установи OmniRoute вручную: `npm install -g omniroute`
2. Запусти: `omniroute`
3. Открой http://localhost:20128
4. Сделай скриншоты нужных страниц
5. Сохрани их в папку `assets\`

## Сценарии тестирования

### Сценарий 1: Чистая установка (идеальный случай)

**Предусловия:**
- Node.js НЕ установлен
- OmniRoute НЕ установлен
- Claude Code НЕ установлен
- Есть интернет-соединение
- Скриншоты в папке `assets\`

**Шаги:**
1. Запусти `ClaudeCodeInstaller.ahk`
2. Дождись автоматической установки Node.js
3. Дождись автоматической установки OmniRoute
4. Когда откроется окно "Подключение провайдера Kiro":
   - Перейди в браузер
   - Нажми "Connect" на провайдере Kiro
   - Войди через Google
   - Вернись в окно установщика
   - Нажми "Готово"
5. Когда откроется окно "Создание API ключа":
   - Перейди в браузер
   - Нажми "Create API Key"
   - **СКОПИРУЙ ключ в буфер обмена** (Ctrl+C)
   - Вернись в окно установщика
   - Нажми "Готово"
6. Дождись завершения установки
7. Открой новый терминал
8. Введи команду: `CC`
9. Проверь, что Claude Code запустился

**Ожидаемый результат:**
- Все шаги выполнены автоматически
- API ключ взят из буфера обмена
- CC.bat создан в C:\Scripts\
- Claude Code запускается командой `CC`

### Сценарий 3: Забыл скопировать API ключ

**Предусловия:**
- Стандартные

**Шаги:**
1. Запусти `ClaudeCodeInstaller.ahk`
2. На шаге "Создание API ключа":
   - Создай ключ в браузере
   - **НЕ копируй его в буфер обмена**
   - Нажми "Готово" в установщике
3. Должно появиться окно InputBox
4. Вставь ключ вручную
5. Продолжи установку

**Ожидаемый результат:**
- Установщик обнаружил пустой буфер обмена
- Показал InputBox для ручного ввода
- Ключ успешно сохранён в CC.bat

## Проверка после установки

### 1. Проверь файл CC.bat

```bash
cat C:\Scripts\CC.bat
```

**Должно быть:**
```batch
@echo off
set ANTHROPIC_BASE_URL=http://localhost:20128/v1
set ANTHROPIC_AUTH_TOKEN=<твой_ключ>
set ANTHROPIC_API_KEY=
set ANTHROPIC_MODEL=kr/claude-sonnet-4.5
set ANTHROPIC_SMALL_FAST_MODEL=kr/claude-sonnet-4.5
set CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
claude %*
```

### 2. Проверь PATH

```bash
echo %PATH%
```

Должен содержать: `C:\Scripts`

### 3. Проверь установленные пакеты

```bash
npm list -g omniroute
npm list -g @anthropic-ai/claude-code
node --version
```

### 4. Запусти Claude Code

```bash
# Открой новый терминал!
cd D:\Yandex.Disk\Sync\Coding\Launch Claude Code
CC
```

## Известные проблемы и решения

### Проблема: Node.js установлен, но не виден

**Решение:**
- Перезапусти компьютер
- Или перезапусти explorer.exe
- Или открой новый терминал

### Проблема: OmniRoute не запускается

**Решение:**
- Проверь, что порт 20128 свободен: `netstat -ano | findstr 20128`
- Запусти вручную: `omniroute`

### Проблема: Claude Code не запускается

**Решение:**
- Проверь, что OmniRoute запущен
- Проверь API ключ в C:\Scripts\CC.bat
- Проверь логи установщика

## Логи и отладка

### Где смотреть логи:

1. **Лог установщика** - в окне установщика (нижняя часть)
2. **Лог OmniRoute** - в терминале, где запущен omniroute
3. **Лог Claude Code** - в терминале, где запущен CC

### Если что-то пошло не так:

1. Сделай скриншот окна установщика с логом
2. Скопируй текст из лога
3. Проверь, на каком шаге остановилась установка
4. Попробуй выполнить этот шаг вручную
