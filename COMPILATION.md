# Инструкция по компиляции ClaudeCodeLauncher.exe

## Требования

- **AutoHotkey v2+** должен быть установлен
- Компилятор **Ahk2Exe** (входит в состав AutoHotkey v2)

## Важно: Параметры компиляции

**Обязательные требования:**
- **Архитектура:** 32-bit (x86) - для максимальной совместимости
- **Иконка:** `Assets/LCC.ico` (устанавливается автоматически через директиву)

## Метод 1: Через контекстное меню (рекомендуется)

1. Найдите файл [`ClaudeCodeLauncher.ahk`](ClaudeCodeLauncher.ahk:1) в проводнике Windows
2. Нажмите правой кнопкой мыши на файл
3. Выберите **"Compile Script"** (Компилировать скрипт)
4. Компилятор автоматически создаст `ClaudeCodeLauncher.exe` в той же папке

**Важно:** Убедитесь, что компилятор настроен на 32-bit режим (по умолчанию в AutoHotkey v2).

**Примечание:** Компилятор автоматически использует директивы из скрипта:
- `;@Ahk2Exe-SetName` - имя приложения
- `;@Ahk2Exe-SetDescription` - описание
- `;@Ahk2Exe-SetVersion` - версия (текущая: 1.3.4)
- `;@Ahk2Exe-SetMainIcon` - иконка приложения (`Assets\LCC.ico`)

## Метод 2: Через GUI компилятора

1. Запустите **Ahk2Exe.exe** из папки установки AutoHotkey
   - Обычно: `C:\Program Files\AutoHotkey\Compiler\Ahk2Exe.exe`
2. В поле **"Source (script file)"** укажите путь к `ClaudeCodeLauncher.ahk`
3. В поле **"Destination (.exe file)"** укажите `ClaudeCodeLauncher.exe`
4. В поле **"Base File"** выберите **"AutoHotkey32.exe"** (32-bit версия)
5. Нажмите **"Convert"**

**Примечание:** Иконка будет установлена автоматически из директивы `;@Ahk2Exe-SetMainIcon`.

## Метод 3: Через командную строку

```cmd
"C:\Program Files\AutoHotkey\Compiler\Ahk2Exe.exe" /in "ClaudeCodeLauncher.ahk" /out "ClaudeCodeLauncher.exe" /base "C:\Program Files\AutoHotkey\v2\AutoHotkey32.exe"
```

**Параметры:**
- `/in` - исходный .ahk файл
- `/out` - выходной .exe файл
- `/base` - базовый файл AutoHotkey (32-bit версия)

**Примечание:** Иконка устанавливается автоматически через директиву в коде, указывать `/icon` не требуется.

## Проверка версии после компиляции

После компиляции проверьте версию:

1. Нажмите правой кнопкой на `ClaudeCodeLauncher.exe`
2. Выберите **"Свойства"**
3. Перейдите на вкладку **"Подробно"**
4. Проверьте поле **"Версия файла"** - должно быть **1.3.4**

## Директивы компиляции в коде

В файле [`ClaudeCodeLauncher.ahk`](ClaudeCodeLauncher.ahk:9) используются следующие директивы:

```ahk
;@Ahk2Exe-SetName Claude Code Launcher
;@Ahk2Exe-SetDescription Лаунчер для Claude Code через Omniroute
;@Ahk2Exe-SetVersion 1.3.4
;@Ahk2Exe-SetMainIcon Assets\LCC.ico
```

Эти директивы автоматически применяются при компиляции любым методом, поэтому иконку указывать вручную не требуется.

## Секция IgnoreBegin/IgnoreEnd

Код между директивами `;@Ahk2Exe-IgnoreBegin` и `;@Ahk2Exe-IgnoreEnd` (строки 16-20) выполняется только при запуске скрипта, но игнорируется при компиляции. Это используется для установки иконки в трее при разработке.

## Troubleshooting

**Проблема:** Нет пункта "Compile Script" в контекстном меню
- **Решение:** Переустановите AutoHotkey v2 с опцией установки контекстного меню

**Проблема:** Ошибка компиляции "Cannot find Ahk2Exe.exe"
- **Решение:** Убедитесь, что AutoHotkey v2 установлен полностью, включая компилятор

**Проблема:** Скомпилированный .exe не запускается
- **Решение:** Проверьте, что в коде нет синтаксических ошибок. Запустите сначала .ahk файл для проверки.

## После компиляции

1. Протестируйте скомпилированный `ClaudeCodeLauncher.exe`
2. Проверьте, что версия в заголовке окна отображается как **v1.3.4**
3. Убедитесь, что все функции работают корректно
4. Создайте бэкап скомпилированного .exe в папку `backup/` (опционально)
